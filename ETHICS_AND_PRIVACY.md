# AI Ethics and Privacy for CookieCloudyDay

## 1. Transparency

Demi เป็นผู้ช่วย AI ของร้าน CookieCloudyDay ดังนั้นผู้ใช้ควรรู้ว่ากำลังคุยกับ AI ไม่ใช่เจ้าของร้านโดยตรง

แนวทางที่ใช้:
- ใช้ชื่อว่า “Demi ผู้ช่วย AI ของร้าน CookieCloudyDay”
- ไม่แอบอ้างว่าเป็นเจ้าของร้าน
- ใช้ไอคอน/ข้อความที่ทำให้รู้ว่าเป็นผู้ช่วย AI
- ถ้าคำตอบอาจไม่ครบถ้วน ควรให้ลูกค้าติดต่อร้านโดยตรง

## 2. Hallucination and Responsibility

AI อาจแต่งข้อมูลเองได้ ถ้าไม่มีข้อมูลใน knowledge base เช่น ราคา เมนู หรือข้อมูล allergy

แนวทางป้องกัน:
- ให้ chatbot ตอบจาก knowledge base ของร้านเป็นหลัก
- ถ้าไม่มีข้อมูล ให้ตอบว่าไม่ทราบ แทนการแต่งข้อมูลเอง
- คำถามเกี่ยวกับสุขภาพหรือ allergy ควรแนะนำให้ลูกค้าสอบถามร้านโดยตรง
- ไม่ให้ AI รับประกันข้อมูลที่อาจมีผลต่อความปลอดภัยของลูกค้า

## 3. Data Privacy and PDPA

โปรเจกต์นี้ไม่ควรเก็บข้อมูลส่วนตัวของลูกค้าโดยไม่จำเป็น เช่น เบอร์โทรศัพท์ ที่อยู่ รูปภาพ หรือข้อมูลบัตรประชาชน

แนวทางที่ใช้:
- ไม่ส่งข้อมูลส่วนตัวของลูกค้าเข้า Gemini API หากไม่จำเป็น
- ไม่เก็บข้อมูลส่วนตัวใน Google Sheets แบบ public
- API key และ token ต้องอยู่ใน `.env`, GitHub Secrets หรือ HuggingFace Secrets
- ไม่ commit ไฟล์ `.env`, `.json`, token หรือ secret ลง GitHub
- หากอนาคตมีระบบรับออเดอร์จริง ต้องแจ้ง purpose และขอ consent จากลูกค้า

## 4. Bias and Fairness

AI อาจตอบไม่เป็นธรรมชาติในภาษาไทย หรือแนะนำเมนูไม่เหมาะกับบริบทของลูกค้า

แนวทางป้องกัน:
- ใช้ knowledge base ภาษาไทยของร้าน
- ทดสอบคำถามหลายรูปแบบ เช่น “ขอเมนูหน่อย”, “มีอะไรขายบ้าง”, “ไม่ชอบหวานมากกินอะไรดี”
- หลีกเลี่ยงคำตอบที่ stereotype ลูกค้าตามเพศ อายุ หรือกลุ่มคน
- ให้คำตอบเป็นกลางและอ้างอิงจากข้อมูลร้าน

## 5. Cost Awareness

บริการที่เกี่ยวข้องกับโปรเจกต์นี้มี free tier เช่น Gemini API, HuggingFace Spaces, GitHub Actions, Google Sheets และ Telegram Bot

แนวทางควบคุม cost:
- ใช้ Gemini 2.5 Flash เพราะเพียงพอสำหรับโปรเจกต์นี้
- ไม่เรียก API ซ้ำใน loop โดยไม่จำเป็น
- ใช้ RAG เพื่อส่งเฉพาะ context ที่เกี่ยวข้อง ไม่ส่ง knowledge base ทั้งหมดทุกครั้ง
- ใช้ HuggingFace CPU Basic Free
- ไม่ upgrade hardware โดยไม่จำเป็น
- ตรวจ quota และ logs เป็นระยะ

## 6. Decision Framework

ก่อนเพิ่ม feature AI ใหม่ จะใช้คำถาม 3 ข้อนี้:

1. Trust: ถ้า AI ตอบผิด ใครเสียหาย และเสียหายแค่ไหน
2. Cost: ถ้าผู้ใช้เพิ่มขึ้น 10 เท่า ค่าใช้จ่ายจะเพิ่มมากแค่ไหน
3. Privacy: ข้อมูลที่ AI เห็นเป็นข้อมูลส่วนตัวหรือไม่ และถ้าหลุดจะกระทบใคร

ถ้ามี red flag เช่น เกี่ยวกับสุขภาพ ข้อมูลส่วนตัว หรือค่าใช้จ่ายสูง ต้องเพิ่ม guardrail ก่อน deploy

## 7. Summary

CookieCloudyDay ใช้ AI เพื่อช่วยตอบคำถามทั่วไปของร้าน เช่น เมนู ราคา เวลาเปิด และช่องทางสั่งซื้อ โดยออกแบบให้โปร่งใส ไม่แอบอ้างเป็นคนจริง ลด hallucination ด้วย RAG และไม่เก็บข้อมูลส่วนตัวของลูกค้าโดยไม่จำเป็น
